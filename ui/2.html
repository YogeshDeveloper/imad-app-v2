Introduction

Path Tracker is a cloud and mobile based platform. It helps organizations Plan, Analyse and Track its Field Operations (Sales and Service) by enabling Real Time Tracking of its Field Workforce.

Our application helps you create new orders, view all existing orders in a List/Map based view, group and assign them to your agents for processing.

Our mobile application helps guide the field agents with the location of customers with inbuilt map that helps him navigate directly to the desired location. It helps the manager to track and communicate with agent in real time through Mobile App instantaneously

Our mobile application allows customer to track the location of the filed agent and communicate with agent.

We also support integrating our software with other 3rd party software

Assignment List

Assignment List gives you a detailed table that describes the orders that are assigned to an agent and status of the agent.

Whenever an order is assigned to an agent, the status of that assigned order will reflect on Assignment List.

Click Actions Click Refresh to view the latest update on the Assignment List



The image is given for an example, once you start assigning orders to your agents Assignment List table will look like the image.

The columns in the table are Customer Name, Assignment Status, Agent Status, Type, Area.

Explanation for each column in the table are as follows

Customer Name: Name of the customer for whom order is assigned

Assignment Status: Status of the assignment it can be either In Process or Completed (once the agent clicks Complete on his mobile App)

Agent Status: Status of the agent when he starts processing an order It can be ACCEPT (Once the agent clicks Accept on his mobile App) STARTED (once the agent clicks Started on his mobile App) or DONE (once the agent clicks Service Start Complete on his mobile App)

Type: Type of the service that an agent does

It can be service or AMC or Installation or any other service which you mentioned on Service Type while creating an order.

Area: Area where an agent is processing the order

Dashboard

Dashboard is one of the key features of Path Tracker.

It helps you monitor the agents, orders and their exact status on several time periods such as Hourly, Daily, Monthly with the help of Dashboards and reports.

Dashboards and Report gets updated automatically and provide you with exact information.

Dashboard section contains Main Dashboard, Hourly Dashboards, Daily Report and Monthly Report.

Main Dashboard

Main Dashboard has three dashboards. They are Order Status, Order Per assets, Time elapsed since order placed.

Click Dashboard on the side navigation bar and Click Main Dashboard

Order Status



Order status pie chart gives you status of all the orders on the account.



Order status is of three types

In Process - Orders that are assigned and to be completed

Completed - Orders that are assigned and completed

To be Assigned - Orders to be assigned and to be started

Order Per Asset

Order per asset graph explains number of assignments given to the agent



Time elapsed since order placed



Time elapsed since order placed pie chart gives details about number of orders placed in between the time periods



It can be of two types periods

Time elapsed between 1-24 hours

Time elapsed >24 hours

Hourly Dashboard

Hourly Dashboards gives the hourly status of the open and closed orders.

Click Dashboard on the side navigation, Click Hourly Dashboards

There are three types of hourly dashboards. They are

Total Open orders per hour (Last Hour)

Open Vs Closed orders (Prior 24-hour period)

Total open orders per hour

Total open orders per hour graph describes the open orders on a hourly basis. It reflects the exact changes open orders per hour.



Open Vs Closed orders (last hour)

Total open orders and closed orders per hour graph describes the number of open orders and closed orders on hourly basis for 24 hours.

It reflects the exact changes in open and closed orders per hour.



Open vs Closed orders(for prior 24 hour period

Total open orders and closed orders per hour graph describes the number of open orders and closed orders on hourly basis for 24 hours.

It reflects the exact changes in open and closed orders per hour.

Transit(Hours) – Number of hours that an agent has travelled for the day.

Trips – Number of trips that an agent had done for the day.

Distance Travelled – Total distance that an agent had travelled for the day.

Total Orders – Total number of orders that are assigned for an agent.

Open Orders – Total number of orders that an agent had to complete.

Closed Orders – Total number of orders that an agent has completed in a day.
